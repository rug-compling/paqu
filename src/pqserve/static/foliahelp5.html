Gebruik van metadata

Onder metadata definieer je de elementen uit de metadata die je met je corpus wilt invoeren in PaQu.

Alleen definities die je aanvinkt worden uiteindelijk gebruikt.

Bij label geef je de naam op waarmee dit item in de uitvoer komt.

Daaronder moet je aangeven wat voor soort metadata het is. Voor uitleg, zie: Info — metadata.

Tot slot moet je aangeven wat de bron is van de metadata, en de expressie waarmee het element dat je hebben wilt is te vinden.

Optie: Extern XPath
Hier is een voorbeeld van een FoLiA-bestand dat verwijst naar externe metadata: 
<?xml version="1.0" encoding="UTF-8"?>
<FoLiA xmlns="http://ilk.uvt.nl/folia" xmlns:xlink="http://www.w3.org/1999/xlink">
  <metadata src="voorbeeld.xml" type="cmdi">
    ...
  </metadata>
  <text>
    ...
  </text>
</FoLiA>
Je moet een XPath-expressie gebruiken om aan te geven hoe je iets uit de metadata wilt halen. Voor voorbeelden, zie beneden.
Optie: Intern XPath
Hier is een voorbeeld van een FoLiA-bestand met daarin metadata opgenomen: 
<?xml version="1.0" encoding="UTF-8"?>
<FoLiA xmlns="http://ilk.uvt.nl/folia" xmlns:xlink="http://www.w3.org/1999/xlink">
  <metadata>
    <foreign-data xmlns:dc="http://purl.org/dc/elements/1.1/">
      <dc:identifier>mydoc</dc:identifier>
      <dc:format>text/xml</dc:format>
      <dc:type>Example</dc:type>
      <dc:contributor>proycon</dc:contributor>
      <dc:creator>proycon</dc:creator>
      <dc:language>en</dc:language>
      <dc:publisher>RadboudUniversity</dc:publisher>
      <dc:rights>publicDomain</dc:rights>
    </foreign-data>
  </metadata>
  <text>
    ...
  </text>
</FoLiA>
Dit is metadata met een simpele structuur, maar het kan ook complex zijn, zoals bijvoorbeeld cmdi.

Je moet een XPath-expressie gebruiken om aan te geven hoe je iets uit de metadata wilt halen. Voor voorbeelden, zie beneden.

Optie: Native ID
Hier is een voorbeeld van native metadata: 
<?xml version="1.0" encoding="UTF-8"?>
<FoLiA xmlns="http://ilk.uvt.nl/folia" xmlns:xlink="http://www.w3.org/1999/xlink">
  <metadata type="native">
    <meta id="titel">De avonden</meta>
    <meta id="auteur">Gerard Reve</meta>
  </metadata>
  <text>
    ...
  </text>
</FoLiA>
Je gebruikt de waarde van het ID om de metadata te selecteren, hier titel of auteur.

Dat is een hoop informatie. Je wilt maar een of enkele elementen eruit halen. Dat kun je met een XPath-expressie doen.

De titel: 

    CMD/Components/Session/Title
Het land: 
    CMD/Components/Session/MDGroup/Location/Country
Hier een voorbeeld dat gebruik maakt van de interne metadata, zoals in het voorbeeld onder het kopje Intern XPath. De root van de metadata is voor interne metadata altijd foreign-data: 
    foreign-data/publisher
Let op: Gebruik geen namespaces in je XPath-expressies.

Let op: Begin je expressie niet met een enkele schuine streep.
Beginnen met een dubbele schuine streep kan wel, zoals: //MDGroup/Location/Country

Voorbeelden van XPath met speaker
Je kunt FoLiA gebruiken voor het weergeven van dialogen. Met het attribuut speaker geef je dan aan wie wat zegt. Bijvoorbeeld: 
    <s speaker="jaap"><t>Hoe is het?</t></s>
    <s speaker="mien"><t>Ik mag niet klagen.</t></s>
Wanneer je de string %speaker% (dus tussen procenten) in je XPath gebruikt, dan wordt dat vervangen door de actuele waarde van het attribuut speaker, en zo kun je een waarde voor de juiste spreker uit de metadata halen.

Voorbeeld, de leeftijd van de huidige spreker: 

    CMD/Components/Session/MDGroup/Actors/Actor[Code='%speaker%']/Age
Voorbeeld, de eerste taal van de spreker: 
    CMD/Components/Session/MDGroup/Actors/Actor[Code='%speaker%']/Keys/Key[@Name='CGN.firstLang']
Als je [Code='%speaker%'] zou weglaten, dan zou je bij elke zin de gegevens van alle sprekers krijgen.
